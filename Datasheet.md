Part 1: Datasheet for the BBO Capstone Project Dataset
Motivation
I created this dataset to benchmark derivative-free optimization strategies on eight complex black-box functions. Each function simulates a unique, real-world challenge like locating a 2D radiation source or tuning 8D hyperparameters where you cannot access the underlying mathematical formulas. The dataset supports optimization research, specifically testing how well algorithms find global peaks when working with a highly restricted evaluation budget. It provides a direct history of how sequential, human-guided decisions interact with multi-dimensional landscapes over time. It was developed independently for my university capstone project without external backing or funding.
Composition
The dataset consists of eight distinct sub-functions packaged into a unified numerical framework. Each instance contains a multi-dimensional coordinate vector (X) and its corresponding scalar maximization score (Y).
Functions 1 & 2 (2D): 20 instances each (10 baseline, 10 sequential queries)
Function 3 (3D): 25 instances total (15 baseline, 10 sequential queries)
Functions 4 & 5 (4D): 40 and 30 instances respectively
Function 6 (5D): 30 instances total
Functions 7 & 8 (6D & 8D): 40 and 50 instances respectively
All data is structured as raw floating-point arrays originally stored in NumPy .npy formats.
There is a severe, intentional sampling bias within this composition. Because the assignment set a tight limit of 10 rounds, I had to stop exploring the maps early on and spend most of my query budget aggressively exploiting narrow, high-performing pockets. Consequently, there are massive geographical gaps in the data where large regions of the search spaces remain completely blank and unsampled. The dataset contains entirely synthetic, simulated numbers, meaning there is zero human data, private information, or identifiable tracking involved.
Collection Process
The collection process took place in two distinct stages. It began with static, pre-generated baseline samples used to establish a rough map of the terrain. Following the baseline, I spent 10 consecutive weeks sequentially submitting exactly one new query coordinate per function each round.
My sampling strategy was completely data-driven: I used a systematic, step-halving bisection rule to narrow down coordinate intervals whenever a point yielded a better output score. If a query tanked, I reversed direction along that axis. In Week 9, I executed a drastic "emergency pivot" for Functions 7 and 8 after recognizing that my local search loops were trapped in sub-optimal valleys. I abandoned the local paths entirely and made a macro-leap back to the coordinates of the highest initial sample benchmarks (Index 6 and Index 14). No ethical reviews or consent protocols were necessary since the data is purely synthetic math.
Preprocessing and Uses
I chose not to apply any filtering, normalization, or standard data-cleaning techniques to this dataset. The input coordinates sit naturally within scaled unit boundaries ([0,1]^D), and I preserved the raw Y outputs exactly as the simulation gave them to me. This leaves the raw noise, extreme sparsity, and sudden drops fully intact so optimization models cannot cheat by smoothing out the landscape.
This dataset is perfect for evaluating the sample-efficiency of active learning or Bayesian optimization algorithms under strict iteration limits. However, it is entirely inappropriate for training standard, general-purpose neural networks. Because the sequential query process caused data points to bunch up heavily around a few local peaks, any predictive machine learning model trained on this data will overfit horribly to those tiny, isolated neighborhoods.
Distribution and Maintenance
The dataset is hosted inside my institutional learning management platform and a private GitHub repository used for formal assignment grading. It is restricted strictly to non-commercial academic review and research, meaning there are no usage fees or intellectual property restrictions attached. I am solely responsible for the upkeep of this data.
